### 3.6 API詳細規範

#### 3.6.1 健康檢測API

**獲取資料源健康狀態 (GET /api/v1/health-status/{data_source_id})**

*響應示例：*
```json
{
  "data_source_id": "ds-123",
  "status": "healthy",
  "last_check": "2025-10-31T10:00:00Z",
  "response_time_ms": 45,
  "uptime_7d": 99.8,
  "error_rate": 0.002
}
```

#### 3.6.2 告警API

**创建告警规则 (POST /api/v1/alert-rules)**

告警规则用於定義何時触发告警。支援基於響應時間、可用性等多個指標的複杂规则。
